Как начать работать с bhyve

Последнее изменение: 15/07/2019 17:21:45

Почему bhyve?

  1. Бесплатно
  2. Доступен из коробки во FreeBSD
  3. Не надо разбираться с гуями
  4. С обёрткой из скриптов vm-bhyve с ним приятно работать
  5. Гостями можно и *BSD, и GNU/Linux, и Windows, и даже больше

Аппаратное обеспечение

Для работы bhyve требуется наличие CPU Intel, поддерживающих механизм виртуализации VT-x и работу со вложенными страницами памяти (EPT — Extended Page Tables). Опционально для проброса PCI-устройств в госетвые окружения может быть задействована технология VT-d (Direct Device Attach). Проверить наличие VT-x можно просмотрев флаги в секции "Features" в выводе dmesg, в списке должен присутствовать флаг VMX.

Я буду использовать сервер HP ProLiant DL360p Gen8 с одним четырехъядерным процессором Intel Xeon E5-2609, восемью гигабайтами оперативной памяти и с двумя жесткими дисками в зеркале.

Куда жать?

Сперва установим операционную систему FreeBSD 12.0, настроим сеть, скачаем и распакуем последние обновления (этот процесс я описывать не буду), затем приступим к настройке bhyve и установке прикладного ПО.

Работу гипервизора обеспечивает загружаемый модуль ядра vmm.ko, библиотека libvmmapi и утилиты bhyve, bhyveload и bhyvectl. Дополнительно мы установим пакет vm-bhyve для управления гипервизором.

 # pkg update
 # pkg install vm-bhyve
