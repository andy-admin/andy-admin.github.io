Ввод в эксплуатацию сервера под управлением FreeBSD

Последнее изменение: 25/07/2019 16:30:55

Во время установки системы:

  1. указываем полное имя с доменной частью, должно совпадать с dns-именем
  2. в приоритете схема разметки GPT (UEFI)
  3. ФС UFS или ZFS, если это необходимо
  4. отказаться от UTC, использовать местный часовой пояс
  5. создать пользователя отличного от root с классом russian, включить его в группу wheel

Сразу после установки:

  1. сетевые интерфейсы конфигурируем по DHCP
  2. для авторизации по ключу-ssh копируем последний на сервер
  3. обновляемся, используя команду freebsd-update
  4. обновляем менеджер пакетов и пакетную базу
  5. синхронизируем время командой ntpdate
  6. настраиваем alias для рута, указываем почту, куда будем ждать отчеты periodic
  7. ставим munin-node, выпиливаем ненужные плагины
  8. ставим прикладуху, настраиваем
  9. в крон задания на создание резервных копий и загрузку их на удаленный сервер-хранилище

Периодически:

  1. устанавливаем бинарные обновления
  2. обновляем ПО из пакетов
  3. проверяем наличие бэкапов и возможность восстановления из них
