Как использовать cloud images

Последнее изменение: 14/11/2019 17:44:32

В сети легко найти готовые образы виртуальных машин с развернутой системой FreeBSD. Также легко их можно использовать для создания новых гостей, сразу же запуская их в работу. Образ должен быть в формате RAW или QCOW2. Чтобы использовать эту функцию, вам нужно установить пакет qemu-utils:

 root@vmm1:~ # pkg install qemu-utils

Команда vm img загрузит образ в хранилище данных и при необходимости распакует его (поддерживаются файлы .xz, .tar.gz и .gz):

 root@vmm1:~ # vm img https://download.freebsd.org/ftp/releases/VM-IMAGES/12.1-RELEASE/\
 amd64/Latest/FreeBSD-12.1-RELEASE-amd64.raw.xz

Для создания новой виртуальной машины из образа выполните:

 root@vmm1:~ # vm create -i FreeBSD-12.1-RELEASE-amd64.raw fbsd121

Запустив машину, выполните действия:

  1. смените пароль рута
  2. настройте hostname
  3. sshd в автозагрузку
  4. установите часовой пояс
  5. ntpdate в автозагрузку
  6. заведите пользователя в группе wheel
  7. cкопируйте на машину ключ для авторизации по ssh
  8. обновите систему, установив последние патчи
  9. обновите менеджер пакетов и пакетную базу
  10. установите пакет munin-node и создайте симлинки на нужные плагины
  11. установите другие пакеты в соответствие условиям задачи, настройте ПО и введите сервер в эусплуатацию

Чтобы просмотреть загруженные образы выполните команду vm img.

 root@vmm1:~ # vm img
 DATASTORE           FILENAME
 default             FreeBSD-12.0-RELEASE-amd64.raw
 default             FreeBSD-12.1-RELEASE-amd64.raw

Для удаления образа из хранилища и списка достаточно удалить файл образа из каталога /vm/.img.
